# Teaching Adaptation Examples

## Learning Style Patterns

### theory-first

- Start with principles, constraints, and terminology.
- Add one concrete example after the mental model is stable.

### practice-first

- Start with a small task the junior can complete quickly.
- Explain the abstraction after the first success.

### question-driven

- Build frequent check-in prompts.
- Ask the junior to restate tradeoffs in their own words.

### observation-first

- Demonstrate one full example.
- Then switch to guided repetition with smaller hints each round.

### trial-and-error

- Offer constrained experiments.
- Focus debrief on what changed and why.

## Review Coaching Examples

- If weakness is `slows down on ambiguity`, convert broad review notes into ordered next actions.
- If weakness is `hesitates to ship early drafts`, explicitly separate "draft acceptable" from "production-ready".
- If strength is `written communication`, ask for short implementation notes or PR summaries.
