# Chat Summarizer

チャット履歴を「AIへの引き継ぎ書」として構造化するスキル。
目的は「このmdを貼り付けるだけで、別のAIが即座に同じ文脈で会話を再開できる状態」を作ること。

---

## 原理：何が「良い引き継ぎ書」か

人間同士の引き継ぎと同じで、受け取った側が「なぜその決断をしたか」まで理解できることが重要。
単なる会話ログではなく、**意図・決定・状態・次のアクション**の4つが揃っていれば再開できる。

| 要素 | 意味 | 欠けると |
|------|------|---------|
| 意図 | なぜこの会話を始めたか | 方向性がずれる |
| 決定 | 何を選び、何を却下したか | 同じ議論を繰り返す |
| 状態 | 現時点で何が完成しているか | どこから再開すべか分からない |
| 次のアクション | 次に何をすべきか | 受け取ったAIが迷う |

---

## ワークフロー

### Step 1: 会話を分析する

現在のチャット履歴全体を読み、以下を抽出する：

- **トピック**: 何について話しているか
- **ゴール**: ユーザーが最終的に達成したいこと
- **決定事項**: 採用した方針・技術選択・設計決定
- **却下事項**: 検討したが採用しなかった選択肢とその理由
- **現在の状態**: 完成済みのもの・作業中のもの・未着手のもの
- **成果物**: 生成したコード・ファイル・コマンド等
- **次のアクション**: 続きとして何をすべきか

### Step 2: Markdownを生成する

以下のテンプレートで出力する。
**各セクションは受け取ったAIが即座に理解できる粒度で記述すること。**

```markdown
# 引き継ぎ: {トピックの一言要約}

> 生成日時: {datetime}  
> 元の会話の目的: {ユーザーが最初に何をしたかったか}

---

## 🎯 ゴール

{最終的に達成したいこと。箇条書き可。}

---

## ✅ 決定事項

{採用した方針・技術・設計を列挙。なぜそれを選んだかの理由も記載。}

- **{決定1}**: {理由}
- **{決定2}**: {理由}

---

## ❌ 却下した選択肢

{検討したが採用しなかったものと理由。これがあると同じ議論の繰り返しを防げる。}

- **{選択肢1}**: {却下理由}

---

## 📦 現在の状態

### 完成済み
{すでに動いている・作成済みのもの}

### 作業中
{途中まで進んでいるもの}

### 未着手
{これからやる予定だったもの}

---

## 📁 成果物

{生成したコード・ファイル・コマンドなどを記載。重要なコードスニペットはそのまま貼る。}

```{language}
{code}
```

---

## ⏭️ 次のアクション

{続きのAIが最初にすべきこと。具体的な指示として書く。}

1. {アクション1}
2. {アクション2}

---

## 🗒️ 補足・注意事項

{ハマりポイント・前提条件・環境情報など、知っておくべきこと}
```

### Step 3: ファイルに書き出す

```bash
# ファイル名は日時+トピックで自動生成
{topic_slug}_{YYYYMMDD_HHMM}.md

# 出力先
/mnt/user-data/outputs/{filename}
```

### Step 4: ユーザーに渡す方法を案内する

ファイルを `present_files` で提示し、以下を添えて案内する：

- **Claude Codeに渡す場合**: `cat {filename}.md | pbcopy` でコピーして新しいセッションの最初に貼り付ける
- **新しいチャットに渡す場合**: ファイルを添付するか、内容をそのまま貼り付ける
- **Cursor / Copilotに渡す場合**: `.md` ファイルをプロジェクトルートに置いてコンテキストとして読み込む

---

## 品質チェックリスト

生成したmdを出力する前に以下を確認する：

- [ ] 「なぜその決定をしたか」が読み取れるか
- [ ] 却下した選択肢が記録されているか（同じ議論の防止）
- [ ] コードスニペットは実際に使ったものがそのまま入っているか
- [ ] 次のアクションが「具体的な動詞」で書かれているか（「検討する」ではなく「〇〇を実装する」）
- [ ] 受け取ったAIが追加質問なしで作業を再開できるか

---

## 引き継ぎ先別の調整

| 引き継ぎ先 | 調整ポイント |
|-----------|------------|
| Claude Code | コマンド・ファイルパスを具体的に。環境情報（OS・言語バージョン）を必ず記載 |
| 新しいClaudeチャット | 口語的な補足を加えて読みやすく。System promptが引き継がれないことを考慮 |
| ChatGPT / Gemini | Claude固有の記法（XMLタグ等）を使わず、汎用的な記述にする |
| Cursor / Copilot | コードの文脈を厚めに。ファイル構成・依存関係を明示 |
