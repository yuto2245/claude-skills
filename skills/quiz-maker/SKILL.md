# Quiz Maker スキル

選択形式の問題を `ask_user_input_v0` を使って出題する。

---

## ask_user_input_v0 を使う理由

`ask_user_input_v0` はClaudeの標準ウィジェット機能で、選択肢をボタンUIとして表示できる。
テキスト入力不要で回答でき、回答後すぐにフィードバックを返せるため、問題出題に最適。

---

## 問題の出題方法

### 単一選択（4択）

```
ask_user_input_v0({
  questions: [{
    question: "【問題】〈問題文〉",
    type: "single_select",
    options: ["A: 〈選択肢〉", "B: 〈選択肢〉", "C: 〈選択肢〉", "D: 〈選択肢〉"]
  }]
})
```

- 選択肢は4つ固定（A/B/C/Dのラベルを付ける）
- 正解の位置は毎回変える（常にAが正解にならないよう注意）
- ひっかけ選択肢には実務・試験でよくある誤解を使う

### 複数選択（複数正解あり）

```
ask_user_input_v0({
  questions: [{
    question: "【問題】〈問題文〉（正しいものをすべて選んでください）",
    type: "multi_select",
    options: ["A: 〈選択肢〉", "B: 〈選択肢〉", "C: 〈選択肢〉", "D: 〈選択肢〉"]
  }]
})
```

- SAP認定試験など「2つ選べ」形式の問題に使う

---

## 回答後のフィードバック

ユーザーが回答したら即座にフィードバックを返す。

**正解の場合**
```
✅ 正解です！

【解説】
〈なぜ正しいのかを原理から説明。実務との関連も添える〉
```

**不正解の場合**
```
❌ 不正解です。正解は「〈正解〉」です。

【なぜ間違いか】
〈選んだ選択肢が誤りである理由を具体的に説明〉

【正解の解説】
〈正解が正しい理由を原理から説明〉
```

---

## 次の問題への誘導

フィードバックの後は必ず以下を `ask_user_input_v0` で提示する。

```
ask_user_input_v0({
  questions: [{
    question: "次はどうしますか？",
    type: "single_select",
    options: ["次の問題へ", "この概念をもっと詳しく教えて", "終了する"]
  }]
})
```

---

## 問題品質の基準

| 項目 | 基準 |
|------|------|
| 問題文 | 50〜150文字。何を問うかを明確に |
| 選択肢 | 各20〜60文字。紛らわしすぎず、かつ簡単すぎない |
| 正解の根拠 | 公式仕様・実務事例に基づく |
| ひっかけ | よくある誤解・混同しやすい概念を使う |
