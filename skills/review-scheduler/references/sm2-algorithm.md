# SM-2 アルゴリズム仕様

## 原理

人間の記憶は指数的に減衰する（エビングハウスの忘却曲線）。
しかし、適切なタイミングで復習するたびに減衰カーブが緩やかになる。

SM-2はこの性質を利用して「忘れかける直前」に復習をスケジュールする。
つまり、復習が成功するたびに次の間隔を**指数的に伸ばす**ことで、
最小の復習回数で最大の定着を狙う。

```
記憶保持率
100% ┤ ╲    ╲      ╲         ╲
     │  ╲    ╲      ╲          ╲
 50% ┤   ╲    ╲      ╲           ╲
     │    ╲     ╲       ╲            ╲
  0% ┼────┴──────┴────────┴──────────────→ 時間
     1日  6日   前回×EF    前回×EF
     ↑     ↑      ↑          ↑
    復習1  復習2   復習3      復習4
```

## 各カードが保持する状態

| 変数 | 初期値 | 意味 |
|------|--------|------|
| repetitions (n) | 0 | 連続正解回数 |
| ease_factor (EF) | 2.5 | そのカードの「覚えやすさ」。高いほど間隔が速く伸びる |
| interval (I) | 0 | 次の復習までの日数 |
| next_due | 登録日 | 次に復習すべき日付 |

## アルゴリズム本体

```
入力: quality (q) = 0〜5 のユーザー自己評価
       n, EF, I = 前回の状態

【Step 1: 間隔の計算】
if q >= 3 (正解):
    if n == 0:  I = 1      ← 初回は翌日
    if n == 1:  I = 6      ← 2回目は6日後
    if n >= 2:  I = round(I × EF)  ← 3回目以降は前回間隔 × EF
    n = n + 1

if q < 3 (不正解):
    n = 0       ← リセット
    I = 1       ← 翌日からやり直し

【Step 2: EFの更新】（正解・不正解どちらでも実行）
EF = EF + (0.1 - (5 - q) × (0.08 + (5 - q) × 0.02))
if EF < 1.3: EF = 1.3   ← 下限

【Step 3: 次回日付】
next_due = today + I日

出力: n, EF, I, next_due
```

## quality（品質スコア）の意味

Claudeのチャット文脈では、フラッシュカードのような明確な正解/不正解がない。
代わりに、以下の簡略化した4段階を使う：

| quality | ラベル | 意味 | 使い方 |
|---------|--------|------|--------|
| 5 | 完璧 | すぐに思い出せた | ユーザーが「覚えてる」「もう大丈夫」と言った |
| 4 | 良好 | 少し考えたが正解 | ユーザーが概要は覚えているが詳細があいまい |
| 3 | ギリギリ | かなり苦労して思い出した | ユーザーが「なんだっけ？」と聞いた後に思い出した |
| 1 | 忘却 | 全く思い出せない | ユーザーが「全然覚えてない」と言った |

## EF更新式の直感的理解

`EF = EF + (0.1 - (5 - q) × (0.08 + (5 - q) × 0.02))`

これは二次関数で、qが高いほどEFが上がり、低いほど下がる：

| q | EFの変化量 | 意味 |
|---|-----------|------|
| 5 | +0.10 | 簡単 → 間隔をもっと伸ばしていい |
| 4 | +0.00 | ちょうどいい → 現状維持 |
| 3 | -0.14 | ギリギリ → 間隔を少し縮める |
| 2 | -0.32 | 不正解 → かなり縮める |
| 1 | -0.54 | 完全忘却 → 大幅に縮める |
| 0 | -0.80 | ブラックアウト → 最大限縮める |

つまりEFは「このカードの難易度を反映する係数」。
簡単なカードは指数的に間隔が伸び、難しいカードは短い間隔で繰り返される。

## Notion DBプロパティへのマッピング（sm2_compact方式）

SM-2の4状態を1プロパティに圧縮して保存する。
トークン効率のため、個別のNumberプロパティではなく `sm2_compact` を使用。

| SM-2変数 | 保存先 | 形式 |
|----------|--------|------|
| repetitions | sm2_compact内 `n` | n0, n1, n2... |
| ease_factor | sm2_compact内 `ef` | ef2.5, ef2.36... |
| interval | sm2_compact内 `i` | i0, i1, i6, i15... |
| quality | sm2_compact内 `q` | q1〜q5, 未評価は `q-` |
| next_due | sm2_next_due（独立プロパティ） | Date型（Notionビュー用） |

**sm2_compact の例**: `n3|ef2.36|i15|q4`

sm2_next_due だけ独立プロパティにしている理由：
Notionのカレンダービューとフィルタ機能が Date 型プロパティに依存するため。
これにより「Notion上で視覚的に復習予定を確認」と「Claudeの低トークン読み取り」を両立する。
