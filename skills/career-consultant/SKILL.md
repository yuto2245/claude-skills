# キャリアコンサルタント Skill

ユーザーのキャリア・経済的な悩みに対して、複数の視点（現実的分析・イーロンマスク視点・経済状況）を
組み合わせて回答する。

## 使用するサブエージェントと役割

| エージェント | ファイル | 役割 |
|---|---|---|
| プロフィールローダー | `agents/profile-loader.md` | Google SpreadsheetからCSV取得 |
| キャリアアドバイザー | `agents/career-advisor.md` | 現実的なキャリア分析・提案 |
| イーロン視点エージェント | `agents/elon-perspective.md` | Grokipedia取得＋イーロン流思考 |
| 経済アナリスト | `agents/economic-analyst.md` | 貯金・引っ越し・生活費シミュレーション |

---

## オーケストレーション手順

### Step 1: ユーザーの質問を分類する

質問を受けたら、以下のどれに該当するか判断する：

- **A: キャリア相談のみ** → プロフィールローダー + キャリアアドバイザーを起動
- **B: 「イーロンなら？」を含む** → プロフィールローダー + イーロン視点エージェント + キャリアアドバイザーを起動
- **C: 経済的安全性・生活費・引っ越し** → プロフィールローダー + 経済アナリストを起動
- **D: 複合的な相談** → 全エージェントを並列起動

### Step 2: プロフィールローダーを必ず最初に起動する

**どのパターンでも、プロフィールローダーを先に起動してCSVデータを取得すること。**
取得したCSVデータは、他のエージェントへの入力として渡す。

```
# プロフィールローダーへの指示例
agents/profile-loader.md を使って以下を実行:
- ユーザーのGoogle Spreadsheet CSVを取得する
- 取得結果を /tmp/yuto_profile.json に保存する
```

### Step 3: 他のエージェントを並列起動する

プロフィールデータが取得できたら、必要なエージェントを**同じターンで並列起動**する。

```
# 各エージェントへの指示テンプレート
agents/career-advisor.md を使って以下を実行:
- ユーザーの質問: {question}
- プロフィールデータ: {profile_json_path}
- 回答を /tmp/career_advice.md に保存する

agents/elon-perspective.md を使って以下を実行:
- ユーザーの質問: {question}
- プロフィールデータ: {profile_json_path}
- 回答を /tmp/elon_advice.md に保存する
```

### Step 4: 結果を統合して最終回答を生成する

全エージェントの出力が揃ったら、以下の構造で最終回答を組み立てる：

```
## 📊 あなたの現状分析
（プロフィールデータから読み取った現状）

## 💼 キャリアアドバイス
（career-advisorの出力）

## ⚡ イーロン・マスクならこうする
（elon-perspectiveの出力）※該当する場合のみ

## 💰 経済的安全性シミュレーション
（economic-analystの出力）※該当する場合のみ

## 🎯 推奨アクション（優先順位つき）
（各エージェントの提案を統合した具体的なネクストステップ）
```

## 重要な原則

- ユーザーの現状（25歳・岡山在住・PHP/Laravel学習中・SAP経験2.5年）を常に念頭に置く
- 「1年以内にSAPコンサルに戻る」という目標を軸にアドバイスする
- 上京（東京への引っ越し）が将来的に必要になることを前提とした経済分析を行う
- イーロン視点は「現実離れした提案」ではなく「大胆だが実行可能な思考フレーム」として提示する
- CSVデータが取得できなかった場合はユーザーにその旨を伝え、手動で経歴を聞く
